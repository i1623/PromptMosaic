## 基本概念

| 用語 | 意味 |
| --- | --- |
| **タイル** | プロンプトの最小単位。1 つの単語・文章に対応し、ON/OFF・強調・並べ替えができます。 |
| **二言語表示** | 英語プロンプトと日本語などの表示名を並べて表示する仕組み。PromptMosaic の中心的な用途の 1 つです。 |
| **翻訳支援** | LM Studio などのローカル LLM を使い、日本語の単語や文章を英語プロンプト用タイルへ変換する任意機能。 |
| **ブロック** | タイルを束ねる入れ物。標準で Positive / Negative などの役割を持ちます。 |
| **生成テンプレート** | InvokeAI の txt2img ワークフローグラフ。PromptMosaic はここにプロンプト等を差し込んで生成します。**ベースモデルごとに必要**です。 |
| **ベースモデル** | SDXL / SD 1.x / FLUX / FLUX2 / Anima など、モデルの種別。テンプレートはベース単位で管理されます。 |
| **系譜 / 履歴マップ** | 生成の親子関係。ある生成から派生した生成をツリーで辿れます。 |
| **継承権者（heir）** | 「次の生成」が子として記録される対象ノード。 |
| **プラン** | 複数モデル／LoRA／パラメータを一度に巡回生成する設定。 |

> ⚠️ **重要:** 画像生成そのものは InvokeAI が行います。PromptMosaic は InvokeAI のキューへ生成を送る補助ツールです。

## 画面構成

```
┌──────────────┬─────────────────────────────────┬──────────────┐
│ 左ペイン        │        中央ペイン                    │ 右ペイン        │
│ ◀ タグ         │  ┌ 系譜ナビ（親カード/継承権者） ─┐    │ 履歴 ▶         │
│              │  │ 🗺️履歴マップ ✂開祖 ↩↪      │    │              │
│ [タグ]        │  └────────────────────────┘    │ [履歴]        │
│ [モデル]       │   Positive ブロック               │ [ノート]       │
│ [LoRA]       │     □ tile  □ tile  □ tile       │ [ゴミ箱]       │
│ [文章]        │   Negative ブロック               │              │
│              │     □ tile                       │              │
│              │  ── 生成バー ──                    │              │
│              │  モデル/解像度/Steps/CFG/seed/Count  │              │
│              │  [履歴生成][1️⃣][単純生成]            │              │
└──────────────┴─────────────────────────────────┴──────────────┘
```

ペインは上部のツールバーのボタンで開閉できます。

- `◀ タグ` — 左ペイン開閉
- `履歴 ▶` — 右ペイン開閉
- `⚙ 設定` — 設定ダイアログ
- `🔀 シャッフル` — シャッフル ON のブロックのタグ順を再シャッフル
- `↩ / ↪` — 元に戻す / やり直す（中央ペイン編集）
- **オールクリア** — LoRA とすべてのブロックを初期化（Positive / Negative の保存グループがあれば既定として再配置）

## 左ペイン（ブラウザ）

左ペインはタブで 4 つのモードを切り替えます。共通して、項目を中央ペインへ**ドラッグ＆ドロップ**して使います。

### タグブラウザ

`🏷 タグ` タブ。タグをカテゴリ別に管理します。

- タグを中央ペインのブロックへドラッグして追加。
- **カテゴリはユーザーが自由に増減でき、辞書ごとに独立**しています。
- **辞書**（タグの集合）はリネーム・コピー・削除・エクスポート / インポートが可能。
- 横断検索（`🔍 横断検索`）でタグ / グループ / 文章をまたいで検索できます。
- タグ入力欄では候補サジェストを表示できます（設定でオン/オフ）。

> 📷 *スクリーンショット差し込み位置: タグブラウザ*
> `docs/images/tag_browser.png`

### モデルブラウザ

`🧠 モデル` タブ。InvokeAI から同期したモデルを管理します。

- モデルを**ダブルクリックで選択**。
- **Invoke と同期** ボタンで最新のモデル / LoRA 一覧を取得。
- 各モデルにサムネイル・1 行メモ・詳細メモを設定可能。
- **デフォルト生成パラメータ**（Steps / CFG / Scheduler）をモデル単位で登録でき、選択時に自動適用。
- **トリガーワードセット**・**ネガティブプロンプトセット**・**自動読み込み LoRA** をモデルに紐付け可能。
- NSFW フラグ付きモデルは設定で一覧から隠せます。
- 「すべて表示」OFF 時は、サムネイル/コメントを設定済みのモデルだけ表示。

### LoRA ブラウザ

`🎨 LoRA` タブ。LoRA を 14 + 1 の大分類ジャンルで管理します。

- LoRA をダブルクリックで追加 / 再度ダブルクリックで解除。
- **weight**（適用強度）を個別に編集。
- LoRA ごとにサムネイル・コメント・トリガーワードを注釈付け。
- ジャンル（キャラクター / 衣装 / 画風 / 光・色 など）は追加・リネーム・削除が可能。
- 選択中モデルのベースに**非対応の LoRA は自動的に除去**され、通知されます。
- 履歴で使った LoRA は「履歴 LoRA」として参照できます。

> ⚠️ LoRA はベース依存です。モデルのベースを変えると非対応 LoRA は外れます。

### 文章プロンプト / タイルグループ

`文章` タブ・**タイルグループ**ブラウザ。

- **文章プロンプト** — よく使う長文（自然文）プロンプトをカテゴリ別に保存・呼び出し。
- **タイルグループ** — タイルの組み合わせをひとまとめに保存し、ワンクリックで中央へ展開。
- どちらもカテゴリの追加・リネーム・削除・移動に対応。

## 中央ペイン（プロンプトエディタ）

### ブロックとタイル

中央ペインはプロンプトの本体です。**ブロック**（Positive / Negative / Top / Middle / Bottom などの役割）の中に**タイル**を並べます。

ブロックヘッダーの操作（一部は右クリックメニュー）:

- **🔓/🔒 ロック** — ロック中は半透明オーバーレイで覆われ、タイル操作・入力を遮断（誤編集防止）。
- **シャッフル** — そのブロックのタグ順をシャッフル対象にする。
- **🔀** ボタンでシャッフル ON のブロックを再シャッフル。
- 右クリック → **全展開 / 全畳む / 現地語コピー / ブロッククリア**。

### タイルの操作

| 操作 | 効果 |
| --- | --- |
| ドラッグ | 並べ替え |
| クリック（トグル） | ON / OFF（一時的に除外） |
| 強調 `＋ / －` | そのタグの効き（emphasis）を増減 |
| `▲ / ▼` | 上下に移動 |
| 削除 | タイルを除去 |
| ダブルクリック / 編集 | タグ名・表示名・本文テキストの編集 |

ブロック下部の入力欄では、`masterpiece, 1girl, blue eyes:1.2` のようにカンマ区切りで複数タグを追加できます。**文章追加** は入力を分割せず、1 つの文章タイルとして追加します。

**タイル表示は 3 段階で巡回**できます（ボタンで切替）。

1. 2 段（現地語 + プロンプト）
2. 現地語 1 段
3. プロンプト 1 段

### 翻訳機能

ローカル LLM（LM Studio）を設定しておくと、ブロック入力欄から翻訳タイルを追加できます。

- **単語翻訳追加** — 1 単語を翻訳してタイル化。
- **文章翻訳追加** — 自然文を翻訳してタイル化。
- 翻訳に使う LLM・翻訳プロンプトは設定 / フローティングエディタで調整可能。
- **LLM 自動分類** — タグを LLM でカテゴリ分類する補助機能（任意）。

> 翻訳・分類は**任意機能**です。LM Studio 未設定でも PromptMosaic の生成自体は使えます。

## 生成バー

中央ペイン下部。ここで送信内容を決めます。

### モデルとベース

- **ベースモデルコンボ** — 「単一」モデルか「プラン」かを選びます。
  - `単一` … 1 つのモデルで生成。
  - `プラン` … [マルチモデルプラン](#マルチモデルプラン)で複数モデルを巡回。プラン選択時は「プラン編集」ボタンが出ます。
- モデルの変更は左ペイン **🧠 モデル** タブでダブルクリック。
- モデル切替時、モデルに登録された既定パラメータがあれば無言で適用されます。

### 生成パラメータ

| 項目 | 説明 |
| --- | --- |
| **幅 / 高さ** | 出力解像度（px、8 の倍数に自動補正） |
| **Steps** | 生成ステップ数 |
| **CFG** | プロンプト忠実度（1.0 以上。蒸留モデルでは 1.0 固定・編集不可） |
| **Scheduler** | スケジューラ（モデルのベースに応じて選択肢が制限されます） |
| **Count** | 1 回の生成で作る枚数（runs） |
| **InvokeAI での保存先** | InvokeAI 側の保存ボード。括弧内は現在の画像数。 |

### シード設定

- **シード値** — 数値指定（ランダム無効時に使用）。
- **🎲** — ランダムに採番。
- **ランダムシード** チェック — 枚ごとにランダム。
- **🔓 / 🔒 シード固定トグル** — 4 つの挙動:

| トグル | ランダム | 挙動 |
| --- | --- | --- |
| 🔓 | OFF | 指定値から連番 |
| 🔓 | ON | 枚ごとにランダム |
| 🔒 | OFF | 全枚を同じ固定値で生成 |
| 🔒 | ON | 生成回ごとに 1 つ採番し、その回の全枚で固定 |

> ⚠️ **変動要素ガード:** 🔒固定 × Count ≥ 2 で、プロンプトに変動要素（ランダム/シーケンシャルのグループ、シャッフル）が無い場合、「同じ画像を量産するだけ」になるため生成を中止して警告します。

### 3 つの生成ボタン

| ボタン | 履歴記録 | 用途 |
| --- | --- | --- |
| **履歴生成** | あり | 標準。横のチェックで記録方法を調整。 |
| **1️⃣ 履歴 1 件生成** | 代表 1 件のみ | Count 枚を生成し、代表 1 件だけ履歴に残す。 |
| **単純生成** | なし | InvokeAI へ送るだけ。履歴を残さない。 |

**履歴生成の横のチェックボックス:**

- **マップ** — ON で、生成を現在の基点の子として履歴マップにも残す。
- **1 件** — ON で、Count 枚生成しても最初の 1 件だけ履歴に残す。

**全件記録の確認:** Count が 10 以上のとき、「全件記録 / 履歴 1 件 / 記録なし / キャンセル」の 4 択ダイアログが出ます（履歴ツリーが過大になるのを防ぐため）。

**生成の中止:** 送信中は ⏹ で中止できます（未送信分を破棄し、未受信の画像をキャンセル）。

## 右ペイン（サイドパネル）

`履歴 ▶` で開閉。タブは **履歴 / ノート / ゴミ箱**。

### 生成履歴

- 生成済みの画像がサムネイル付きで一覧表示されます（直近順）。
- **ダブルクリック**でプロンプト + 全パラメータを中央ペインに復元。
- 右クリックメニュー:
  - **⬆ 全てロード（プロンプト + 全パラメータ）** / **⬆ プロンプトのみロード**
  - **編集…**（レビューダイアログ）
  - **📁 他のグループへ移動 / 新規グループ作成**
  - **🗑 ゴミ箱へ移動**
  - **📂 画像を開く / 📁 画像のフォルダを開く**
  - **🔄 サムネ更新**
  - **現在の系譜へ接続**
- **星評価**（★）をクリックで付与。お気に入り絞り込み・ステータス絞り込み・検索が可能。

> 📷 *スクリーンショット差し込み位置: 右ペイン履歴*
> `docs/images/side_panel_history.png`

### 履歴の編集（レビュー）

履歴を右クリック →「編集…」で **履歴の編集ダイアログ**が開きます。

- **プロンプト** タブ — Positive / Negative / シード / サイズ / Steps / CFG / Scheduler / ベースモデル / モデル / テンプレートを確認。
- **編集** タブ — お気に入り・評価（★）・ステータス（下書き / 候補 / 承認済み / 却下 / アーカイブ）・1 行コメント・メモを記録。
- **中央ペインにロード** で復元。サムネイル変更も可能。

### グループ管理

履歴はフォルダ（グループ）で階層管理できます。

- 新規グループ作成（ルート / この階層 / サブグループ）。
- グループ編集で**名前・保存先フォルダ・NSFW フォルダ**を設定。保存先は親から継承可能。
- グループごとゴミ箱へ / グループだけ削除（中身は上の階層へ）。

### ノート

`📓 日記ノート` タブ。日付ごとに気づきや試行錯誤を Markdown で記録できます。

### ゴミ箱

削除した履歴の退避先。

- ゴミ箱から復元可能。
- **完全削除（選択）** / **ゴミ箱を空にする**（元に戻せません）。

## 生成系譜（系譜ナビゲーター）

中央ペイン上部のカード群で、現在の生成位置（系譜）を把握・操作します。

- **親カード** — 現在ノードの親。クリックで親の設定をロード（シードは除く）。
- **継承権者カード** — 「次の生成」が子として記録される対象。`◀ / ▶` で同じ親の兄弟を切り替え、中央ペインのタイル ON/OFF もその記録に追従します。
- **✂ 開祖ボタン** — 系譜を切り、このノードを開祖（新しいルート）にします。
- **🗺️ 履歴マップ** — 拡大履歴マップを開きます。
- 継承権者カードのサムネ / 番号をクリックすると一時的な拡大プレビューを表示。

> 「系譜なし」のときは、次の生成が新しい開祖（ルート）になります。

## 履歴マップ（パラレルワールド）

`🗺️` で開く、生成の親子関係を可視化するツリーです。広大な生成履歴を俯瞰し、過去の任意の時点へジャンプできます。

> 📷 *スクリーンショット差し込み位置: 履歴マップ全体*
> `docs/images/history_map_full.png`

### 操作の原則

- **左クリック = 便利系**（見る・スクロール・ズーム）
- **右クリック = 現状を変える操作**（メニュー・展開）

### 表示・移動

| 操作 | 効果 |
| --- | --- |
| ノードを左クリック | 画像ビューアでプレビュー |
| ホイール | ズーム（カーソル中心） |
| 左 / 中ドラッグ（余白・線上） | パン（移動） |
| `📍` | 現在地へ直行 |
| `⛶` | ツリー全体にフィット |
| `1:1` | 100% 表示 |
| 現在地 | 常時回転発光で強調 |

### 整理操作

| 操作 | 効果 |
| --- | --- |
| `Shift + 左ドラッグ` | 範囲選択 |
| `Ctrl + 左ドラッグ` | ノードを直系祖先へドロップして親付け替え（傍系・無関係へは拒否） |
| 右クリック → **編集** | 履歴の編集ダイアログ |
| 右クリック → **ここ以下のみ表示 / 全体表示** | 表示範囲を絞る / 戻す |
| 右クリック → **別系統にして移動** | その枝を別系統として切り離す |
| 右クリック → **消去** | ノードを取り除き子を親へ接続（ゴミ箱なし・警告なし） |
| 右クリック → **削除** | 消去 + 履歴をゴミ箱へ（警告あり） |
| 範囲選択 → 一括消去 / 一括削除 | 範囲内だけ消し、範囲外の子は祖先へ繋ぎ直し |

`Esc` はドラッグのキャンセルに使い、マップは閉じません。

### 画像ビューア

ノードを左クリックで開く**履歴画像ビューア**:

- `▲ ▼ ◀ ▶` で直系 / 傍系をナビ移動（履歴マップ側の対象も連動）。
- 左ドラッグで画像移動、ホイールで倍率変更。
- **フィット / 原寸大 / 手動倍率** の 3 状態。
- **中央ペインに反映** ボタンでその生成をロード。
- 閉じるボタン以外では閉じません。
- ウィンドウ位置・サイズは記憶され、設定からリセット可能。

### 色のカスタマイズ

右クリックメニューから**履歴背景色 / 系統線の色 / 文字色**を変更できます（テーマ別の既定は設定からも調整可能）。

## マルチモデルプラン

複数のモデル・LoRA・パラメータを **1 回の生成で巡回**する機能です。

設定方法:

1. 生成バーのベースモデルコンボで **「プラン」** を選択 → **プラン編集** ボタン。
2. プランダイアログで**行（モデル単位）**を追加します。各行に設定するもの:
   - **モデル / ベース / Scheduler / Steps / CFG / 枚数 / LoRA**
   - **有効** チェック（その行を生成対象にするか）
   - **個別追加プロンプト**（この行だけに足す Positive / Negative）
3. 行は上下に並べ替え可能。`Default` プランは常に存在します。

生成すると、有効な各行が順次 InvokeAI へ送信されます。

- プラン行の枚数に Count を掛けた合計枚数が生成されます（確認ダイアログあり。「Count を 1 にして実行」も選べます）。
- LoRA はプラン行内でコンボのサジェスト / 絞り込みに対応。有効な LoRA だけが送信され、登録済みのトリガーワード等は重複を避けて末尾に自動追加されます。

> 📷 *スクリーンショット差し込み位置: マルチモデルプラン編集ダイアログ*
> `docs/images/plan_dialog.png`

プランは **設定 → 生成管理 → マルチモデルプラン管理** からも管理できます。

## 生成テンプレート

テンプレートは InvokeAI の txt2img ワークフローグラフそのものです。PromptMosaic はこれを保存し、生成時に既知のフィールド（プロンプト / シード / Steps / CFG / Scheduler / モデル / 解像度…）だけを差し替えて送信します。

### 取得方法

- **InvokeAI データ取得ウィザード**（初回自動表示 / 設定 → InvokeAI データ取得を開く）

### 仕組みと注意

- **ベースモデルごとに 1 つ以上必要**。同じベースに複数登録でき、生成時に選べます（★ = ベース既定）。
- **取得時は LoRA を 1 つ以上使った生成が必要**です。PromptMosaic は LoRA 経路を見本として再利用するためです。
- VAE / リファイナー / テキストエンコーダーなどの設定違いは**別テンプレートとして名前で区別**します。
- リファイナー段（SDXL 限定の任意機能）はテンプレートに焼き込まれた値を保持し、ベース段にだけパラメータが適用されます。
- InvokeAI のマイナー更新の多くは**テンプレートを取り直すだけ**で吸収されます。
- テンプレート管理で **複製 / 名前変更 / 既定に設定 / 🗑 削除** が可能。テンプレートキャッシュのリセットも可能で、次回生成時に再取得されます。

> ⚠️ 別の InvokeAI 環境の古いテンプレートは動作保証されません。**自分の現在の InvokeAI 環境**から取得してください。

## データ管理

### バックアップ

PromptMosaic のバックアップは、専用ファイルを作る方式ではなく、`data` フォルダを丸ごとコピーする方式です。

- **バックアップ** — アプリを終了してから、`data` フォルダ全体を別の場所へコピーします。
- **リストア** — アプリを終了してから、コピーしておいた `data` フォルダを元の場所へ戻します。
- `index.db` と `suggestions.db` はキャッシュなので、なくても起動時に再構築されます。

> 画像本体を別フォルダで管理している場合は、その画像フォルダも必要に応じて別途コピーしてください。

### タグデータのエクスポート / インポート

タグブラウザのタグ・グループを JSON / CSV で入出力。

- **エクスポート** — 現在のタグとグループを JSON に書き出し。
- **差分追加** — ファイル内の新規タグ・グループのみ追加（既存は変更しない）。
- **上書き更新** — 新規追加 + 既存の同名も上書き。

### キャッシュ管理

- **インデックス再構築** — `data` フォルダをスキャンして `index.db` を更新。
- **サジェスト再構築** — 全ライブラリから `suggestions.db` を再構築。
- `index.db` と `suggestions.db` はキャッシュなので、削除・再構築しても元データは失われません。

### バックアップの基本

アプリ起動中に DB ファイルをコピーすると不完全になる可能性があります。バックアップやリストアの前には PromptMosaic を終了してください。

## 付録

### 外部受信箱

外部ツールから届いた履歴候補を取り込む仕組みです。届いていると起動時 / 更新時に取り込み確認が表示されます。キャンセルしても受信箱に残り、次回再確認されます。

### PNG メタデータの読み込み

PNG をアプリにドロップすると、InvokeAI 形式（`invokeai_metadata` / `invokeai` / `sd-metadata`）のメタデータからプロンプトを読み込めます。

### データベース構成

データは `data/` 以下の複数の SQLite DB に分割されています（`app.db` / `environment.db` / `notes.db` / `history_*.db` / `library_*.db` ほか）。`index.db` と `suggestions.db` はキャッシュで、無くても起動時に再構築されます。

### 起動 / トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| 仮想環境が見つからない | `install_windows.bat` を先に実行 |
| Qt の DLL エラー | Conda/Anaconda の Python を避け、python.org 版で `.venv` を作り直す |
| 生成ボタンが押せない | テンプレート未取得 → InvokeAI データ取得を実行 / モデル・プラン未選択 / プロンプトが空 |
| InvokeAI: 未接続（🔴） | InvokeAI 6.13 以降を起動し、設定 → 接続の URL / キュー ID を確認 |
| テンプレート取得が失敗 | InvokeAI 側で LoRA を使った txt2img を 1 枚生成してから取得 |

ステータス表示: `🟢 接続中` / `🔴 未接続` / `⚪ 確認中…`。
